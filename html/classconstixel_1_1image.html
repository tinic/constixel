Detailed Description

template<template< size_t, size_t, int32_t, bool > class T, size_t W, size_t H, int32_t S = 1, bool GR = false>
class constixel::image< T, W, H, S, GR >

Core class of constixel, holds a buffer of an image of a certain size and format.

Template Parameters
TType of the image buffer. One of format_1bit, format_2bit, format_4bit or format_8bit
WWidth in pixels
HHeight in pixels
SScale factor for sixel output
GRboolean to indicate if palette should be grayscale

◆ device_format

template<template< size_t, size_t, int32_t, bool > class T, size_t W, size_t H, int32_t S = 1, bool GR = false>
enum constixel::image::device_format

Data formats for the convert function

Enumerator
STRAIGHT_THROUGH 

Just copy the data as is.

X_LEFT_TO_RIGHT_1BIT 

1-bit pixel data is stored from left to right, each byte containing 8 pixel values in the x direction.

Y_TOP_TO_BOTTOM_1BIT 

1-bit pixel data is stored from top to bottom, each byte containing 8 pixel values in the y direction.

RGB565_8BIT_SERIAL 

RGB565 pixel data is stored from left to right, each two bytes containing 1 pixel value in the x direction.

RGB666_8BIT_SERIAL_1 

RGB565 pixel data is stored from left to right, each three bytes containing 1 pixel values in the x direction.

RGB666_8BIT_SERIAL_2 

RGB565 pixel data is stored from left to right, each three bytes containing 1 pixel values in the x direction.

template<template< size_t, size_t, int32_t, bool > class T, size_t W, size_t H, int32_t S = 1, bool GR = false>
template<device_format dst_format>
bool constixel::image< T, W, H, S, GR >::convert_chunk ( char * dst,
size_t chunk_size,
size_t & chunk_actual,
size_t & chunk_index )
inline

Convert the current instance into a byte stream formatted for embedded displays. This function will write chunk_length of data into dst and update chunk_index on each call. You should pass chunk_index = 0 at the beginnig of the sequence. Returns true of there is more data, or false if there is no data left. This function is typically used to drive interrupt driven DMA transfers.

Template Parameters
dst_formatThe desired data format.
Parameters
dstThe buffer the data will be written to.
chunk_sizeThe requested chunk size in bytes. This value has to be kept the same during a full conversion sequence.
chunk_actualThe actual amount of bytes which were written into ptr during this call of the function.
chunk_indexThis value will increment after each call of the function. Has to be set to 0 on the first call.
Returns
true if there is more data to convert, false if we reached the end.
